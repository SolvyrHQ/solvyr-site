Sample output

What a Fast Corpus artifact should make inspectable.

This page describes the expected artifact shape. A real sample bundle still needs to be produced from a representative run before this page becomes proof rather than specification.

JSONL records

Source document ID, chunk ID, extracted text, chunk boundaries, embedding reference or vector field, and source metadata.

Run report

Submitted documents, accepted documents, pages, chunks, successful outputs, failures, retries, runtime, caveats, and pricing basis.

Failure notes

Items that could not be processed should be visible instead of silently missing from downstream corpus use.

Still needed

A real downloadable sample bundle.

The next proof upgrade is a small safe run that can publish a sanitized input sample, output JSONL, source manifest, and run report. Until then, this page is intentionally framed as the expected contract, not a finished proof artifact.
